# Admin Portal Improvement

This document outlines recommended improvements and new features for the Admin Portal based on the latest review and audio feedback.

## 1. Inbox / Messages Panel
Currently, the inbox only has "Read" and "Delete" options. It should be expanded with the following features:
- **Reply Functionality:** Add buttons to reply directly via email or provide an in-app reply option.
- **Priority & Organization:** Star priority options, tags, and custom labels.
- **Spam & Management:** Spam blocking options and an Archive feature.
- **Advanced Search & Filters:** Provide search filters to sort by date, time, read status, unread status, name, and email.

## 2. Content Page
- **Search Bar:** Implement a search bar to quickly find content.
- **Live Preview:** Add a "View Live" or "Preview" toggle to preview changes before publishing.
- **UI Proportions:** For pages that have a heading alongside an "Add" section/button, enforce a 60-40 size ratio (60% dedicated to the heading and 40% for the button size).

## 3. Research Papers
- **Management:** Add a search bar and sorting features.
- **Drafting Options:** Add the ability to save research papers as drafts.
- **UI Adjustments:** Follow the 60-40 ratio rule by making the heading larger and the "New Post" buttons relatively smaller.

## 4. Blog Posts
- **Management features:** Add Search, Sort, Draft, and Private options.
- **Drag-and-Drop Reordering:** Replace the manual numbering input system with an intuitive drag-and-drop feature so administrators can visually snap posts into their desired positions.

## 5. Education
- **Reordering:** Implement drag-and-drop functionality to easily arrange educational history records.

## 6. Skills
- **Fix Percentages:** Resolve the issue where the skill proficiency percentages are not being displayed properly.
- **Reordering:** Add drag-and-drop options to reorganize skills.
- **Categorization & Icons:** Group skills by categories and add native icon integration (e.g., displaying the official React icon for "React").
- **Visibility Toggle:** Introduce a visibility toggle. This allows users to add a skill they are actively studying as a "beginner" but easily hide it from the public portfolio until they are proficient enough to show it.

---

*Note: There are more improvements that I would like to suggest, but these are the basic ones which can significantly improve the look and functionality of the admin portal. I will share more changes with you later.*
